HTML and CSS- An Anaolgy

HTML is the structure of the webpage- all the content. The CSS is the styling.

To use a baking analogy: HTML is all the ingredients for our birthday cake- the flour, the sugar, the pan etc. CSS is the decorations- the piping, the sprinkles, the color of the frosting, the position of the happy birthday message and the pretty cake stand.

Control Flow and Loops

Control Flow is how your computer runs its code, typically from the top to the bottom. Like reading a book or a recipe, one line at a time from the first line to the last, unless it runs into some code that changes its flow like a loop.

A loop is like a little break out from the flow. It tells the computer to do something over and over again until certain cicrumcstances are met before it can move on with the rest of the code.

Lets use the steps for making a scrambled egg breakfast as an example of our control flow with stirring the eggs as a little loop that temporarily breaks us out of our flow.

  1. Put the pan on the stove
  2. Turn on the stove
  3. Add oil
  4. Crack Eggs into a bowl
    • Stir eggs
    • Are they mixed? No
    • Stir eggs
    • Are they mixed? No
    • Stir eggs
    • Are they mixed? Yes
  5. Add Eggs to Pan
  6. Cook

What is the DOM?

The DOM, or document object model, is the way through which JavaScript interacts with content within a website-the HTML. It respresents the webpage in a structured, hierarchical way. A very simplified DOM 'tree' is seen below. By using the DOM programmers can easily access and manipulate HTML elements by specifiying an ID, class or element name. One way you can interact with the DOM is by using JavaScript to change an HTML element. Here is an example of this. You could use the following code to select an element with the ID="test"

document.getElementById('test')

And if you wanted to grab the element with the id of 'test' and change its inner HTML text.

document.getElementById('test').innterHTML = "This is my new text"

Data from Arrays and Objects

Arrays and objects are two ways that you can collect data into a group. Lets talk about objects first. An object is used to respresent a thing (for example a person)

let sarah = {
name: "Sarah",
age: 27,
partner: "John"
}

Objects have what is known as a key-value pairs. In the above example "name" would be a key and we can use that key to access the value, or information, stored in it- in this case the name "Sarah". To pull that data out of an object we can use a dot operator followed by the key name (example: sarah.name) or a square bracket notation along with the key in a string formart (example: erin['age']).

Now lets talk about arrays! An array is technically a type of object. The difference is that the data in an array is stored in an ordered collection where you can access the data based on the index location. It is like a sorted list that you can keep A LOT of stuff in.

let favFoods = ['rasberry', 'pasta', 'curry', 'cheese', 'oranges', 'cake']

The 'index' is the location where eat item in the list is store. At [0] is raspberry at [1] is pasta and so on through the whole array. To access the information at a specific location you might do something like this

console.log(favFoods[3]);

This would print out 'cheese' in the console log because cheese is stored at [3]
hint: dont forget arrays start counting from 0, so cheese is the 4th item in the list but is at location [3] in the array.

Functions and Why We Love Them

A function is a block of code that you designed to perform a particular task. You give your function a name and then you execute or 'call' the function as many times as you need. Functions are our heavy-lifters, rather than having to repeat your code multiple times, you can write a function once and reuse it as often as you need.
Here is an example of a very simple function called sayHello that will print out "hello!" in the console.

function sayHello (){
console.log("hello!")
}

sayHello();

So when the function, named sayHello, is called (or executed) "hello!" will print out on the console log.Notice that not only do we need to write the function you also need to call the function.

Functions are very dynamic and can be written to take a paramenter with any data type as the value, they can return a value to you, you can nest functions inside another function and much more.
